# Changelog

## 2.4.2 - 2026-03-19

- 补齐开源发布所需的仓库文档：定位、亮点、安装、启动、截图占位、配置、FAQ 和开源友好说明。
- 新增 `monitor_config.example.json`，方便新用户直接复制本地配置并调整模型别名与计费覆盖。
- 新增主目录 `.gitignore` 和 MIT `LICENSE`，并统一去除个人身份暴露风险。

## 2.4.1 - 2026-03-19

- 将右侧悬浮 drill-down 摘要并入主栅格，改为默认可见的情报面板，消除宽屏时右侧大块留白和漂浮感。
- 修正右侧情报列在 CSS Grid 中被隐式列压窄的问题，让摘要面板与态势侧栏稳定占满整列宽度。
- 保留 hover drill-down 交互，但默认回到实时摘要，不再把关键右侧信息藏在悬浮态里。

## 2.4.0 - 2026-03-19

- 为 Web 监控台加入更高级的实时切换过渡、数值刷新高亮、告警层级动画和悬停 drill-down 面板。
- 新增大屏模式，支持按钮切换和键盘快捷键 `F` / `Esc`，便于长时间挂屏监控。
- 将“累计花费”提升到首屏核心信号位，并保留战术卡中的总费用展示，避免总金额不够显眼。
- 启动输出现在会直接提示如何关闭：前台模式提示 `Ctrl+C`，后台模式提示 `python3 monitor.py stop`，并新增 `close` 别名。
- 收束首屏骨架，减弱重复信息和漂浮面板感，把金额显示改为更克制的监控台排版。

## 2.3.1 - 2026-03-19

- 补强启动链路：后台模式现在会校验现有服务是否健康，不健康时自动重启，避免提示“已运行”但页面实际打不开。
- 新增 `python3 monitor.py open`，用于重新打开已运行的仪表板；如果服务未启动，会自动尝试拉起。
- 后台启动健康检查改为带重试的等待逻辑，失败时明确提示日志路径和前台排查命令。

## 2.3.0 - 2026-03-19

- 进一步重构 Web 监控中心，把首屏从卡片墙收束为更接近指挥台的战术面板、主图焦点和态势解释侧栏。
- 修正 5 小时配额快照选择逻辑，避免模型级 `rate_limits` 快照把更有意义的全局 `codex` 窗口快照覆盖掉。
- 补齐前端与后端监控字段映射，让 5 小时配额、异常等级、活跃模型数、最新活动和窗口粒度使用统一口径。

## 2.2.0 - 2026-03-19

- 重构 Web 监控中心的上半屏信息层级，让 hero、KPI、主图和侧栏更接近高级系统监控台的视觉结构。
- 移除页面中所有会直接展示本机路径来源的内容，包含会话目录、目录来源和工作目录的原始路径输出。
- 将工作区相关展示统一改为匿名化标签，并把终端启动日志里的路径输出改为通用描述，降低开源泄露风险。

## 2.1.1 - 2026-03-19

- 为 Web 监控中心增加进入动画、扫描线氛围、卡片悬浮反馈和表格位移动效，让界面更接近实时监控台。
- 为趋势图、模型柱状图、令牌环图和 5 小时热区条带增加鼠标悬停交互，支持高亮、交叉线和 tooltip。
- 增加 `prefers-reduced-motion` 兼容逻辑，在系统要求减少动画时自动关闭主要动效。

## 2.1.0 - 2026-03-18

- 将 Web 仪表板重构为更接近系统监控中心/NOC 的深色监控台界面，强化窗口健康、实时负载和事件流展示。
- 为最近 5 小时窗口新增 5 分钟滚动槽位聚合 `windows.last_5_hours.by_slot`，让主图和热区条带比按小时聚合更准确。
- 新增 `metrics` 监控指标，包括缓存命中率、输出占比、5 小时与 15 分钟吞吐、活跃模型数、活跃目录数、最近事件新鲜度和异常级别。
- 修正 `rate_limits` 最新快照选择逻辑，新增 `observed_at` 和 `scope` 字段，避免把事件时间错误地与重置时间比较。
- 为事件明细的费率拆分补上 long context 定价修正，确保与主统计口径一致。

## 2.0.1 - 2026-03-18

- 修复 `python3 monitor.py enhanced --once` 因 Rich markup 字符串错误而崩溃的问题。
- 更新内置 OpenAI 模型定价，补充 `gpt-5.4`、`gpt-5.4-mini`、`gpt-5.4-nano`、`gpt-5.4-pro` 等新模型。
- 为 `gpt-5.4` / `gpt-5.4-pro` 增加 long context 定价规则支持。
- 刷新 `gpt-realtime-mini`、`gpt-realtime-1.5`、`gpt-audio-1.5`、`gpt-image-1.5`、`gpt-image-1-mini` 的内置单价。
- 新增 `requirements.txt`，明确增强终端模式所需的 `rich` 依赖。
